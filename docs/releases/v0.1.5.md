# HomeHunt Assistant v0.1.5

## Highlights
- Added in-app onboarding guidance to help first-time users understand viewed/contacted tracking and AI setup.
- Added popup pin reminder tip so users can pin the extension for faster access.
- Improved AI evaluation scoring robustness with a factor-based rubric and calibration to reduce repetitive score clustering.
- Fixed listing price parsing bug where price and change badge values could be concatenated.

## Improvements

### Onboarding and setup
- Added contextual first-run guidance directly in the assistant panel.
- Added AI setup notice with Open Settings CTA when OpenAI API key is missing.
- AI Evaluate button now clearly indicates setup is required when no API key is configured.

### Popup UX
- Added dismissible “pin extension” tip banner in popup.
- Dismissal state is persisted so users don’t see repeated reminders.

### AI scoring quality
- Reworked evaluation prompt/schema to include structured price/quality factor subscores.
- Added local score calibration logic for better spread and consistency.
- Reduced tendency to collapse into narrow score bands (e.g., 50/55/60).

### Bug fixes
- Fixed parser issue that could render listing price incorrectly (e.g., `$4,200` + `$300` change becoming `4,200,300`).
- Added regression test coverage for mixed price/change text parsing.

## Validation
- `npm run typecheck`
- `npm run test`
- `npm run build`
